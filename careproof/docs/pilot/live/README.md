# CareProof Live Pilot Runbook

Use this folder only when a real pilot is active.

## Structure

Create one folder per pilot agency:

`docs/pilot/live/[agency_name]/`

Recommended files inside each agency folder:

- `DAY_17_ISSUE_LOG.md`
- `DAY_17_METRICS.md`
- `DAY_17_CHAMPION_REVIEW.md`
- `DAY_17_CLOSEOUT_EMAIL.md`

## Rules

- Log issues immediately. Do not wait until end of day.
- If any data-isolation or wrong-family/wrong-client issue appears, pause the pilot immediately.
- Track real numbers only. Do not estimate usage after the fact.
- Keep Day 1 scope small enough to support live.

## Launch Priority Order

1. Data safety and security
2. Login and access
3. Visit visibility
4. Check-in and check-out
5. Checklist and care note completion
6. Family update generation
7. Dashboard and reporting
8. Cosmetic UI issues
