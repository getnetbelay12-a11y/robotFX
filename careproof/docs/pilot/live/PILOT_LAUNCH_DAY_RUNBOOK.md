# CareProof Pilot Launch Day Runbook

## Launch Readiness Check

- [ ] Pilot agency account works
- [ ] Admin or champion can log in
- [ ] Caregivers can log in
- [ ] Family members can log in or receive updates
- [ ] Today's visits are scheduled
- [ ] Care plans and checklists are attached
- [ ] Family update method is confirmed
- [ ] Incident workflow has been tested
- [ ] Support contact is confirmed
- [ ] Backup and export plan is understood

## Launch-Day Flow

### Morning

- Confirm all users can log in
- Confirm today's pilot visits
- Confirm support contact

### During Visits

- Monitor check-ins
- Monitor checklist completion
- Monitor notes
- Monitor incidents
- Monitor checkout

### After Visits

- Confirm family updates
- Review dashboard
- Review issues
- Collect pilot champion feedback

## Demo Accounts Pattern

- `owner@careproof.demo / Password123!`
- `caregiver1@careproof.demo / Password123!`
- `family1@careproof.demo / Password123!`

Replace these with real pilot accounts before launch.

## Live Monitoring Checklist

- Caregiver login failures
- Visits not showing
- Wrong caregiver assigned
- Wrong client assigned
- Check-in failures
- GPS permission issues
- Checklist missing
- Care note missing
- Checkout blocked
- Family update not generated
- Incident not appearing in admin
- Family concern not appearing
- Admin dashboard not refreshing

## Brutal Rule

If caregivers cannot log in before visit time, stop and fix that first.
