# CareProof Pilot Handoff

## Pilot Objective

Validate that CareProof improves visit visibility, caregiver documentation, family communication, and operational follow-up.

## Recommended Pilot Scope

- 1 agency branch/team
- 2 coordinators
- 5 caregivers
- 10–20 clients
- 2–4 weeks
- Family portal enabled for selected families

## Pilot Success Metrics

- Visit completion rate
- On-time check-in rate
- Missed visit detection
- Checklist completion rate
- Care notes captured
- Family updates sent
- Concerns submitted and responded to
- Weekly reports sent
- Coordinator follow-up time

## Pilot Setup Steps

1. Configure agency profile.
2. Invite users.
3. Import clients.
4. Add caregivers.
5. Create care plans.
6. Schedule visits.
7. Enable family portal.
8. Train caregivers.
9. Run first week.
10. Review weekly report output.

## Roles And Responsibilities

### Agency Owner

- Approve pilot scope.
- Review pilot outcomes.

### Coordinator

- Manage daily visits.
- Review alerts.
- Respond to concerns.

### Caregiver

- Check in and check out.
- Complete checklist tasks.
- Add notes.
- Report incidents.

### Family

- Receive updates.
- Submit concerns.

### CareProof Admin

- Support setup.
- Monitor pilot.
- Collect feedback.

## Pilot Risks

- Caregivers not using the app consistently
- Incomplete client data
- Family contacts missing
- Care plans not configured
- No internal owner assigned
- Demo mode confused with production

## Go / No-Go Criteria

### Go

- Workflows are stable.
- Users are trained.
- Family visibility rules are reviewed.
- Backup plan is understood.
- Support process is ready.

### No-Go

- Missing client or caregiver data
- Production environment not configured
- Demo reset enabled in production
- Permissions not reviewed
- Family note visibility unsafe
